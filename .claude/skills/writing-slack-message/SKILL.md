# Slack Message Standards

## 必須ルール

### 1. メンションは必ず含める

メッセージには **必ず** 適切な宛先メンションを含めること。メンションなしの投稿は禁止。

- ユーザーから宛先の指示がある場合: その人をメンションする
- 宛先の指示がない場合: ユーザーに「誰にメンションしますか？」と確認してから送信する
- チーム全体への共有: `@channel` または `@here` を使う

メンション対象が不明な場合は `slack_search_users` で検索して特定する。

### 2. 送信相手に応じたメッセージ調整

送信相手の役割・知識レベルに応じて、メッセージの内容・粒度・トーンを適切に変える。

- **技術担当者**: 技術的な詳細（変更内容、影響範囲など）を含めてよい
- **非技術者・上司**: 技術詳細は省き、ビジネスインパクトや結果を中心に伝える
- **外部パートナー**: 丁寧な敬語を使い、背景説明を手厚くする

### 3. 送信相手の調査・提案

ユーザーから宛先を指定されず、自分で `slack_search_users` やチャンネル履歴から送信相手を特定した場合は、送信前に以下をセットでユーザーに提案する:

1. **候補者名**
2. **推定される役割**: チャンネルの直近のやり取りから、その人がPJのメイン担当か、上司・マネージャーか、技術的な知識の程度はどうか、を推測する
3. **メッセージのトーン案**: 上記の役割に基づいて、どの粒度・トーンで書くか

ユーザーの承認を得てから送信すること。

### 4. メッセージ構成

```
<@USER_ID> （メンション）
本文（簡潔に要点を伝える）
```

- 1メッセージ1トピックを原則とする
- 長文は避け、要点を先に書く
- 依頼事項がある場合は明確に書く（「確認お願いします」「レビューお願いします」等）

### 5. スレッド返信

- スレッドに返信する場合は `thread_ts` を必ず指定する
- チャンネル全体にも見せたい場合のみ `reply_broadcast=true` を使う

## フォーマット

Slack の mrkdwn 記法を使用する:

- **太字**: `*text*`
- _イタリック_: `_text_`
- ~取り消し線~: `~text~`
- `コード`: `` `code` ``
- コードブロック: ` ```code``` `
- リンク: `<URL|表示テキスト>`
- メンション: `<@USER_ID>`

## AI臭さの除去（humanizer）

メッセージ本文を作成したら、送信前に `/humanizer` スキルを適用してAI臭さを除去する。

Slackメッセージで特に注意すべきパターン:

- 「確かに」「まさに」「素晴らしい」等の過剰な同調表現を避ける
- 「〜についてですが」「〜の件ですが」のような前置きは最小限にする
- 箇条書きの各項目に太字ヘッダー + コロンのパターンを多用しない
- 絵文字の装飾的な乱用をしない（内容に関係ある場合のみ使用）
- 「ご確認のほど、よろしくお願いいたします」のような定型文を機械的に付けない
- 文末の過剰な丁寧語の積み重ねを避ける（「〜していただけますと幸いです」等）
- 同じ構文の繰り返し（リズムの均一化）を避け、文の長短を自然に混ぜる

送信前の最終チェックとして、作成したメッセージに対して以下を自問する:
- 「この文章がAI生成だと一目でわかるか？」
- わかる場合は humanizer の基準に従って修正してから送信する

## 送信前チェックリスト

送信前に以下を確認:

1. メンション先が含まれているか
2. メッセージの意図が明確か
3. スレッド返信の場合 `thread_ts` が正しいか
4. 機密情報が含まれていないか
5. AI臭い表現が残っていないか（上記 humanizer セクション参照）

## セットアップ

このスキルは `/humanizer` スキルに依存する。未インストールの場合は以下を実行:

```bash
npx skills add https://github.com/blader/humanizer
```

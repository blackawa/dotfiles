# スキルリファクタリング原則

## 何を残すか

**AIが知らない固有の情報だけ残す:**
- このプロジェクト・環境に特有のコマンド・設定
- 失敗しやすい落とし穴とその回避策
- 他スキルで補えない診断コード・テンプレート

## 何を削るか

- AIがすでに知っている一般知識（ツールの基本的な使い方など）
- 他スキルに任せられる内容（例：agent-browserの操作方法 → testing-browserスキルへ）
- 冗長な説明・表・チェックリスト（箇条書きで十分なら箇条書きに）
- ステップの詳細な説明（ステップ名だけで意図が伝わるなら説明不要）

## 目安のサイズ

| スキルの性質 | 目安 |
|---|---|
| 手順・ワークフロー系 | 1–3KB |
| ツール操作系（コマンドリファレンス含む） | 3–6KB |

## 手順

1. 既存の SKILL.md を読む
2. 各セクションに対して「AIが知らない固有情報か？」を問う
3. 削除・圧縮・他スキルへの委譲を決める
4. 書き直して保存
5. サイズを確認（`wc -c SKILL.md`）
